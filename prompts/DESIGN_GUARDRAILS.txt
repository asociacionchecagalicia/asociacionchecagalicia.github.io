
Design Enforcement Instructions
Before generating any code, you MUST follow these locks. If a request conflicts with these, explain the conflict instead of breaking the system.

## 🚨 THE GOLDEN RULE
Do not invent new layout patterns or spacing outside the defined design system.

## Color Usage Rules
- **Text:** Headings and Body MUST be Navy only.
- **Buttons:** Primary background is Coral with White text.
- **Links:** Navy text with underline on hover.
- **Restriction:** Never use Coral or Sky for body text.
- **Contrast:** Maintain 4.5:1 for standard text; 3:1 for large UI components.
 **COLOR LOCK:** Never introduce new colors outside of `docs/BRAND_DNA.md`.

## Typography & Content Rules
- **Paragraphs:** Max width 680px. Never use long centered paragraphs.
- **Headings:** Apply -1% letter-tracking (tracking-tighter).
- **Spacing:** Use only the defined 8px spacing system for margins/padding.

## Component Patterns
- **Buttons:** 8px radius. Padding 12px 20px. Hover state: `translateY(-2px)` + darken 8%.
- **Cards:** White background, 1px soft navy border, 12px radius, soft shadow.
- **Icons:** Line icons only. No fills. Stroke must match logo stroke weight. 24px for UI.

## Visual Motif: Overlapping Circles
- **Limit:** Max 5–7 instances per page.
- **Logic:** 15% minimum overlap. Smallest circle in a pair MUST be Coral.
- **Opacity:** 5%–10% for backgrounds.
- **Exclusion Zones:** Never place behind long text, on buttons, or within 100px of the logo.
- **Usage:** Allowed for hero backgrounds, event cards, and section transitions.

## Interaction Vibe
- The interface must feel soft and breathing. 
- Use generous white space to avoid "Corporate" density.
- Transitions must be calm and structured.

## 🔒 Layout & Design Locks
- **STRUCTURE LOCK:** Never create sections without using `SectionWrapper` and `ContainerWrapper`. Never use `div -> div -> random spacing`.
-- **MOBILE LOCK:** Design mobile-first. Layouts must look perfect at 375px, 768px, and 1200px. 44px minimum tap targets.

# Logo & Header Implementation Guardrails

## 🔒 Logo Application Locks
- **RESPONSIVE LOGO LOCK:** - **Mobile (< 768px):** Use `logo-icon.svg` (Symbol only). 
    - **Tablet (768px - 1199px):** Use `logo-horizontal.svg`.
    - **Desktop (> 1200px):** Use `logo-full.svg`.
- **PLACEMENT LOCK:** The logo must always be wrapped in a link to the homepage (`/`).
- **NAME DISPLAY LOCK:** On mobile, never display the association name in the header. The full name should only appear in the **Hero Heading**, **Footer**, or **About Page**.

## 📱 Mobile Navigation Layout
- **Layout Pattern:** - Left: Menu Toggle (☰) | Center: `logo-icon.svg` | Right: Language Switcher (🌐).
    - Header Total Height: 60px – 72px.
    - Padding: 12px–16px vertical padding.

## ⚡ Technical Asset Rules
- **SVG ONLY:** Always use `.svg` for logos in the UI to ensure retina sharpness and performance.
- **OPTIMIZATION:** Run all SVGs through a cleanup to remove Figma metadata before deployment.
- **FAVICONS:** Ensure `index.html` is updated to reference the full favicon package in `/public/favicons/`.

## 🚨 THE "NEVER" LIST
- Never shrink the full logo text to fit mobile; switch to the icon mark instead.
- Never place the logo motif (overlapping circles) within the logo's 48px/100px clear space.
